# AchieveMe

**AchieveMe** — соцсеть для тех, кому нравится достигать целей в реальной жизни. Визуально и концептуально вдохновлена [Steam Community](https://steamcommunity.com/id/tfmetalstore/): профили, уровни, достижения — только вместо игр здесь **сферы жизни**.

## Концепт

В Steam игры мотивируют через ачивки и уровень профиля. AchieveMe переносит эту механику в реальность:

| Steam | AchieveMe |
|-------|-----------|
| Игры | Сферы жизни (Здоровье, Карьера, Спорт…) |
| Достижения в игре | Жизненные ачивки |
| Часы в игре | Прогресс по сфере (цели и ачивки) |
| Уровень Steam | Уровень профиля (XP за ачивки) |
| Комментарии на профиле | Комментарии на профиле |

Пользователь регистрируется, выбирает сферы, ставит цели, разблокирует достижения — получает XP и растёт в уровне. Другие видят профиль и оставляют комментарии.

## Глоссарий

- **Сфера жизни** — область развития (Здоровье, Карьера и т.д.). Создаётся админом или одобряется из предложений пользователей.
- **Цель** — активная задача внутри сферы, которую пользователь хочет выполнить.
- **Достижение (ачивка)** — разблокированный успех; даёт XP.
- **XP** — опыт, накапливаемый за ачивки.
- **Уровень** — растёт с XP; отображается золотым бейджем на профиле (как в Steam).

### Формула уровней

Для перехода на уровень N нужно `100 × N` XP (накопительно). По умолчанию ачивка даёт 10 XP.

## Стек

- **TypeScript**
- **Next.js 16** (App Router)
- **Prisma 7** + **PostgreSQL**
- **NextAuth.js v5** (email + пароль)
- **CSS Modules** + **BEM**

## Быстрый старт

### 1. Клонирование и зависимости

```bash
npm install
```

### 2. PostgreSQL (Docker)

Скопируйте переменные окружения и поднимите базу:

```bash
cp .env.example .env
docker compose up -d
```

В `.env` уже настроен `DATABASE_URL` под контейнер из `docker-compose.yml`:

```env
DATABASE_URL="postgresql://achieveme:achieveme@localhost:5432/achieveme"
AUTH_SECRET="сгенерируйте: openssl rand -base64 32"
AUTH_URL="http://localhost:3000"
ADMIN_EMAIL="admin@example.com"
ADMIN_PASSWORD="admin123"
```

Остановить базу: `docker compose down`  
Удалить данные: `docker compose down -v`

### 3. Миграции и seed

```bash
npm run db:push
npm run db:seed
```

Seed создаёт:
- **admin** — `admin@example.com` / `admin123` (или `ADMIN_EMAIL` из `.env`)
- **demo** — `demo@example.com` / `demo123` (профиль с ачивками для демо)
- 5 базовых сфер: Здоровье, Карьера, Образование, Спорт, Творчество

### 4. Запуск

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

Демо-профиль: [http://localhost:3000/id/demo](http://localhost:3000/id/demo)

## Маршруты

| Маршрут | Описание |
|---------|----------|
| `/` | Лендинг (редирект на профиль, если авторизован) |
| `/register`, `/login` | Регистрация и вход |
| `/id/[nickname]` | Публичный профиль |
| `/settings` | Редактирование профиля (аватар, баннер, bio) |
| `/achievements` | Достижения, цели, глоссарий |
| `/suggestions` | Предложить сферу + голосование |
| `/admin` | Админ-панель |
| `/admin/spheres` | CRUD сфер |
| `/admin/suggestions` | Модерация предложений |

## Роли

- **USER** — обычный пользователь
- **ADMIN** — управление сферами, модерация предложений

Первый пользователь с email из `ADMIN_EMAIL` получает роль ADMIN при регистрации.

## Голосование за сферы

1. Пользователь предлагает новую сферу на `/suggestions`
2. Другие пользователи голосуют (1 голос на предложение)
3. Админ одобряет или отклоняет на `/admin/suggestions`
4. При одобрении создаётся `LifeSphere`

## UI/UX профиля

Страница `/id/[nickname]` повторяет layout Steam Community:

- Fullscreen-фон (баннер профиля)
- Glass-контейнер с полупрозрачным фоном
- Шапка: аватар 184px, ник, золотой бейдж уровня, XP-бар
- Левая колонка: «О себе», недавняя активность (карточки сфер), комментарии
- Правая колонка: статус, статистика, последние ачивки
